# Support

## Where To Ask

- Use a GitHub issue for reproducible template defects and documentation gaps.
- Use a content-review issue for curriculum or teaching-quality concerns.
- Use private vulnerability reporting for security issues as described in `SECURITY.md`.

## Supported Scope

The latest released major version receives security fixes. The latest minor version receives defect fixes and migration guidance. Generated books own their curriculum, examples, branding and deployment configuration; include the installed template version when reporting a shared-platform issue.

## Service Expectations

This open-source project does not provide a contractual SLA. Maintainers triage security and release-blocking reports first, followed by reproducible defects and enhancement requests. Acknowledgement targets and incident priorities are defined in `docs/internal/operations-runbook.md`.

## Useful Evidence

Include the operating system, Node.js and npm versions, template version, exact command, complete error, minimal reproduction and whether `npm run verify` passes.
