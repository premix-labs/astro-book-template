This repository is a reusable authoring template, not a finished learner-facing book. It demonstrates the reading experience and the workflow used to plan, write, verify, and publish technical tutorials.

<Callout type="tip" title="Recommended start">
  Use <code>npm run create-book</code> instead of copying this folder manually. The initializer removes generated files and template history, resets validation evidence, and updates the new book identity.
</Callout>

## What the Template Includes

- Astro Content Collections with validated MDX frontmatter
- automatic navigation, parts, previous/next links and reading-time estimates
- reusable authoring components and syntax highlighting
- Pagefind search in production output
- light/dark reading themes and responsive navigation
- curriculum, decision, QA and release documents under `docs/internal`
- tests and validators for authoring scripts, chapters and built links

## Before Writing A Real Book

1. Create a new repository with `npm run create-book`.
2. Define the reader, prerequisites, scope, chapter map and final project.
3. Replace the generated introduction.
4. Build the progressive/final examples before claiming a chapter is ready.
5. Run `npm run verify` and update the validation report with real evidence.
