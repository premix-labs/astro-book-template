Validation is evidence, not a confidence statement. Record the exact command, result, date, scope and remaining risk in `docs/internal/validation-report.md`.

## Local Quality Gate

```bash
npm run verify
```

This template runs linting, Astro diagnostics, script tests, book-structure checks, the production build, Pagefind indexing and built-link validation.

## Book-Specific Evidence

The generated book must add checks for its own learner project. Examples include unit tests, integration tests, database migration checks, browser screenshots, accessibility checks or container configuration validation.

<Callout type="warning" title="Do not inherit template evidence">
  A command that passed in this template has not passed in a copied book. The initializer resets validation evidence so every book records its own results.
</Callout>

## Release Gate

A book is not release-ready while required checklist items, progressive examples, final-project builds or critical browser flows remain unverified.

## Checkpoint

Before continuing, confirm that every passing claim in `validation-report.md` names the command, date, scope, and remaining manual risk for the current repository.
