A chapter is complete only when the learner can follow it from the prose. Example projects are verification sources and optional references, not hidden instructions.

## Practical Chapter Shape

<Steps>
1. State the outcome and prerequisites.
2. Explain the problem and each new concept before syntax.
3. List files that will be created or changed.
4. Build the solution in small, located code blocks.
5. Run the relevant command and describe the expected result.
6. Cover common failures and finish with a verifiable checkpoint.
</Steps>

## Code Blocks

Name the file and location before every non-trivial block. Prefer a sequence of focused edits over pasting an entire file. Include PowerShell and Bash commands when operating-system syntax differs.

<Callout type="tip" title="Definition of done">
  The matching progressive example must build or test at this chapter state without files or abstractions from future lessons.
</Callout>
